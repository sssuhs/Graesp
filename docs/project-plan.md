# 项目方案记录

## 1. 项目定位

本项目面向中文硕士毕业设计，目标是实现一套“插排外置式过载监测终端 + PC 上位机”的完整系统。系统通过贴附在插排输入线外皮上的 NTC 热敏电阻采集外部热特征，在不拆开插排、不进入插排内部、不串接电流传感器的条件下，实现电流反演、过载预测、声光报警和数据上传。

## 2. 系统组成

系统分为下位机监测终端和 PC 上位机两部分。

下位机监测终端负责：

- 采集 3 路 NTC 温度信号；
- 对温度数据进行滤波和特征提取；
- 根据温升和温度变化趋势进行电流反演；
- 部署轻量 AI 模型进行过载识别；
- 通过 LED 和蜂鸣器进行本地报警；
- 通过 Wi-Fi 向上位机发送实时数据；
- 采用锂电池供电，并支持 USB-C 充电和下载调试。

PC 上位机负责：

- 通过 Wi-Fi 接收终端数据；
- 显示温度曲线、估计电流、过载概率和报警状态；
- 保存 CSV 实验数据；
- 配置采样周期、报警阈值和算法参数；
- 辅助实验分析和论文结果整理。

## 3. 下位机硬件方案

主控选用 ESP32-S3-WROOM-1-N16R8。该模组具备 16 MB Flash 和 8 MB PSRAM，能够满足 Wi-Fi 通信、数据缓存和轻量 AI 模型部署需求。

下位机硬件模块包括：

- ESP32-S3 最小系统；
- 3 路 NTC 分压采样电路；
- USB-C 供电、充电和下载调试接口；
- 单节锂电池供电模块；
- 3.3 V 稳压模块；
- 电源开关；
- LED 状态指示模块；
- 蜂鸣器报警模块；
- NTC 测温夹具和外置主控盒结构。

## 4. NTC 测温方案

使用 3 路 10 kΩ B3950 NTC 热敏电阻。

建议通道分配：

- NTC1：插排输入线外皮测点 1；
- NTC2：插排输入线外皮测点 2；
- NTC3：环境温度测点。

通过 NTC1 和 NTC2 计算导线外皮平均温度、最高温度和测点差异，通过 NTC3 计算环境温度和稳态温升。

基本特征包括：

- 导线平均温度 `T_wire_avg`；
- 导线最高温度 `T_wire_max`；
- 环境温度 `T_env`；
- 稳态温升 `DeltaT = T_wire_avg - T_env`；
- 两测点温差 `T_diff = abs(NTC1 - NTC2)`；
- 升温速率 `dT/dt`。

## 5. 电源与低功耗

系统采用单节锂电池供电，并通过 USB-C 接口充电。电池输出经电源开关后进入 3.3 V 稳压电路，为 ESP32-S3、NTC 采样电路、LED 和蜂鸣器供电。

低功耗策略包括：

- 周期采样；
- NTC 分压电路按需供电；
- Wi-Fi 周期上传；
- LED 与蜂鸣器仅在状态提示或报警时工作；
- 空闲时进入轻睡眠或低占空比工作状态。

## 6. 数据通信

下位机通过 Wi-Fi 与上位机通信。初期优先采用局域网 TCP 或 WebSocket 方式发送 JSON 数据，上位机解析后进行实时曲线显示和 CSV 保存。

典型数据格式：

```json
{
  "timestamp_ms": 12500,
  "ntc": [58.9, 59.5, 24.7],
  "wire_temp_avg": 59.2,
  "wire_temp_max": 59.5,
  "ambient_temp": 24.7,
  "delta_temp": 34.5,
  "current_est": 12.4,
  "overload_prob": 0.91,
  "state": "warning"
}
```

## 7. 当前不做的内容

- 不做 OLED 显示；
- 不做 SD 卡本地存储；
- 不进入插排内部；
- 不将电流传感器作为终端算法输入；
- 不把实验基准测量模块并入最终终端硬件。
