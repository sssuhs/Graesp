## Why

本项目需要将“插排外置式过载监测终端 + PC 上位机”的毕业设计方案固化为可实现的软件与硬件系统规格。该变更用于明确下位机、上位机、Wi-Fi 通信、NTC 测温、过载判别和报警的边界，避免后续开发中功能范围反复变化。

## What Changes

- 新增 ESP32-S3 下位机监测终端能力：3 路 NTC 温度采集、热特征提取、电流反演、过载判别、LED 与蜂鸣器报警、Wi-Fi 数据上传。
- 新增 PC 上位机能力：Wi-Fi 接收实时数据、曲线显示、参数配置和 CSV 保存。
- 新增上下位机 JSON 通信协议，用于实时监测数据上报和参数下发。
- 新增低功耗与电源管理要求，适配锂电池供电的外置终端使用场景。
- 明确项目非目标：不进入插排内部，不将电流传感器作为终端算法输入，不做 OLED，不做 SD 卡。

## Capabilities

### New Capabilities

- `terminal-monitoring`: ESP32-S3 下位机监测终端的测温、特征计算、电流反演、过载判别、报警和低功耗行为。
- `desktop-supervision`: PC 上位机的数据接收、实时显示、参数配置和 CSV 保存行为。
- `wifi-telemetry-protocol`: 下位机与上位机之间的 Wi-Fi JSON 数据上报和参数下发协议。

### Modified Capabilities

无。

## Impact

- `firmware/`: 需要建立 ESP32-S3 固件工程并实现终端侧功能模块。
- `desktop-app/`: 需要建立 PC 上位机工程并实现数据接收、显示、配置和保存。
- `docs/`: 需要维护项目方案、通信协议、硬件接口和测试说明。
- `hardware/`: 后续需要整理 ESP32-S3、NTC 分压、电源、报警和夹具相关硬件资料。
- `models/`: 后续用于存放过载识别模型、训练脚本和部署文件。
