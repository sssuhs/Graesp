## Context

GraEsp 需要同时实现下位机监测终端和 PC 上位机。下位机硬件基于 ESP32-S3-WROOM-1-N16R8，采用 3 路 10k B3950 NTC 分压采样，其中 2 路用于插排输入线外皮测温，1 路用于环境温度测量。终端使用锂电池供电，通过 LED 和蜂鸣器实现本地报警，通过 Wi-Fi 与 PC 上位机通信。

当前项目刚完成 OpenSpec 初始化，尚未建立固件和上位机代码。设计重点是先形成清晰的系统边界、模块划分和通信协议，后续再逐步实现。

## Goals / Non-Goals

**Goals:**

- 建立下位机和上位机的两级系统架构。
- 明确 ESP32-S3 固件模块：NTC 采样、温度换算、特征提取、电流反演、过载判别、报警、Wi-Fi 通信和低功耗。
- 明确 PC 上位机模块：连接管理、数据解析、实时曲线、参数配置和 CSV 保存。
- 采用 JSON 作为上下位机数据交换格式。
- 支持终端在未连接上位机时独立完成声光报警。

**Non-Goals:**

- 不进入插排内部，不改变插排原有电气结构。
- 不将电流传感器作为终端算法输入。
- 不实现 OLED 显示。
- 不实现 SD 卡本地存储。
- 第一阶段不实现复杂云平台或移动 App。

## Decisions

### 使用 ESP32-S3-WROOM-1-N16R8 作为下位机主控

理由：ESP32-S3 具备 Wi-Fi、较强算力、原生 USB、较充足 Flash 和 PSRAM，适合完成温度采集、边缘 AI 推理、Wi-Fi 数据上传和低功耗控制。相比普通 ESP32，S3 对后续模型部署更稳；相比 STM32，S3 的 Wi-Fi 和上位机联调更方便。

### 使用 3 路 NTC 分压采样

理由：3 路 NTC 能覆盖两个导线外皮测点和一个环境温度测点，既能计算温升，又能通过两测点温差判断贴合差异。相比单点测温，2+1 配置更有鲁棒性；相比 4 路或更多 NTC，3 路方案更容易画板、标定和调试。

### 使用 ESP32-S3 自带 ADC

理由：第一阶段优先降低硬件复杂度和成本。ESP32-S3 自带 ADC 能满足初步测温需求，但需要通过标定降低非线性和分压误差影响。若后续测试发现精度不足，可在硬件迭代中增加外置 ADC。

### 使用 Wi-Fi 作为上下位机主链路

理由：Wi-Fi 适合无线外置终端，减少终端与 PC 的物理连接。USB-C 保留为供电、充电、下载和调试接口，不作为主要数据链路。

### 上位机承担显示和记录

理由：项目不做 OLED 和 SD 卡，终端侧专注采样、判断和报警；上位机负责曲线显示、参数配置和 CSV 保存，使系统分工更清晰。

## Risks / Trade-offs

- ESP32-S3 ADC 精度不足 → 通过 NTC 标定、均值滤波和温度校准降低误差；必要时在后续版本预留外置 ADC 方案。
- Wi-Fi 连接不稳定 → 终端必须在断开上位机时仍能独立报警；上位机需处理断线重连。
- NTC 贴合不稳定 → 通过测温夹具固定传感器位置，并使用两路导线测点计算温差判断异常。
- AI 模型部署复杂 → 第一阶段先实现阈值判别接口，后续替换为轻量 AI 模型。
- 电池续航不足 → 采用周期采样、NTC 分压按需供电、Wi-Fi 周期上传和轻睡眠策略。
