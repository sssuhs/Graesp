# Tasks: 初始终端与上位机系统

## 1. 项目基线

- [ ] 1.1 确认固件开发栈，并建立 ESP32-S3-WROOM-1-N16R8 固件工程。
- [ ] 1.2 确认上位机开发栈，并建立 PC 上位机工程。
- [ ] 1.3 建立统一通信字段定义和示例遥测、配置 JSON 文件。
- [ ] 1.4 整理系统硬件框图、模块清单和 PCB 接口假设。

## 2. 硬件定义

- [ ] 2.1 定义 ESP32-S3 引脚分配，包括 3 路 NTC ADC、LED、蜂鸣器、电池电压检测和调试接口。
- [ ] 2.2 设计 3 路 NTC 分压采样电路，并加入必要的 ADC 输入保护和 RC 滤波预留。
- [ ] 2.3 设计锂电池供电、USB-C 充电、电源开关、3.3 V 稳压和电池电压检测电路。
- [ ] 2.4 设计 LED 与蜂鸣器驱动电路，并记录器件参数和安全工作范围。
- [ ] 2.5 输出第一版 PCB 原理图草稿和测温夹具接口说明。

## 3. 固件采集与特征

- [ ] 3.1 实现 3 路 ADC 采样和 ESP32-S3 ADC 标定。
- [ ] 3.2 实现 NTC 电阻计算和 10k B3950 温度换算，参数应可配置。
- [ ] 3.3 实现 ADC、导线温度、环境温度和温升滤波。
- [ ] 3.4 实现热特征提取，包括平均温度、最高温度、温升、测点温差和升温速率。
- [ ] 3.5 增加串口日志，用于在 Wi-Fi 集成前验证温度读数和特征计算结果。

## 4. 固件过载逻辑

- [ ] 4.1 实现电流反演接口，输入热特征并输出估计电流。
- [ ] 4.2 实现第一版阈值过载检测，支持 `normal`、`warning`、`overload` 状态。
- [ ] 4.3 预留 AI 或轻量模型推理模块边界，后续可替换阈值检测。
- [ ] 4.4 实现 LED 和蜂鸣器在正常、预警、过载状态下的报警策略。
- [ ] 4.5 实现参数保存，至少包括 NTC 参数、报警阈值、采样周期和蜂鸣器开关。

## 5. 固件 Wi-Fi 与功耗

- [ ] 5.1 实现 Wi-Fi station 连接和断线重连。
- [ ] 5.2 实现遥测 JSON 上报，包含设备 ID、时间戳、3 路温度、热特征、估计电流、过载概率、状态和电池电压。
- [ ] 5.3 实现配置 JSON 接收和校验，支持阈值、采样周期、报警开关和标定参数更新。
- [ ] 5.4 实现上位机离线时的本地独立报警行为。
- [ ] 5.5 实现低功耗入口，包括周期采样、Wi-Fi 降占空比和休眠条件判断。

## 6. 上位机应用

- [ ] 6.1 实现 Wi-Fi 数据接收。
- [ ] 6.2 实现 JSON 解析、字段校验和异常数据处理。
- [ ] 6.3 实现 3 路温度、温升、估计电流、过载概率、状态、电池电压和连接状态显示。
- [ ] 6.4 实现导线温度、环境温度、温升和估计电流实时曲线。
- [ ] 6.5 实现参数配置界面，并向终端发送配置 JSON。
- [ ] 6.6 实现 CSV 数据记录，保存时间戳、遥测字段和实验会话信息。
